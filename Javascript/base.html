script의 위치에 따른 차이

  1. head 안에 선언
    • HTML코드를 파싱하다가 head에서 script태그를 만나면 해당 위치의 js파일을 다운 받고 실행한 후 다음 HTML코드를 파싱한다
    • ※ 파싱(Parding) : 컴퓨터에서 컴파일러/번역기가 원시 부호를 기계어로 번역하는 과정
    • js파일의 크기가 크거나 인터넷이 느리다면 js파일을 처리하는데 시간이 오래 걸리므로
    • 웹페이지가 로딩되는데까지 걸리는 시간이 오래 걸린다
  2. body 안 가장 하단 부분에 선언
    • HTML코드를 모두 파싱하고 마지막으로 남은 하단의 js파일들을 다운받고 실행한다
    • 사용자가 기본적인 HTML 컨텐츠를 빨리 확인할 수 있다
    • 웹페이지가 Javascript에 의존적이라면 의미있는 컨텐츠를 보는데까지 오래 걸린다
    • Javascript에 의존적 : JS로 서버데이터를 불러오거나 JS로 DOM요소를 꾸며줌
  3. head안에 async옵션으로 선언
    • HTML코드를 파싱하다가 head에서 script태그를 만나면 파싱과 병렬적으로 js파일을 다운로드하고 다운로드가 끝나면 파싱을 멈춘 뒤 js파일 실행 후 마저 파싱한다
    • body 끝에 선언하는 것보다는 fetch가 병렬적으로 이루어지기 때문에 더 빠르게 로딩된다
    • js파일이 HTML코드가 모두 파싱 되기 전에 실행되기 때문에 후반부에 파싱되는 요소는 아직 정의되어 있지 않아서 조작할 수 없다
    • HTML코드 파싱 도중에 js파일이 실행되기 때문에 js파일 실행시간이 오래 걸린다면 모든 웹페이지 요소가 로딩되기까지 오래 걸린다
    • 다운로드가 먼저 된 js파일부터 실행되기 때문에 순서에 의존적이라면 문제가 발생할 수 있다
  4. head안에 defer옵션으로 선언
    • HTML코드를 파싱하다가 head에서 script태그를 만나면 파싱과 병렬적으로 js파일을 다운로드하고 파싱이 끝나면 다운로드해놨던 js파일들을 실행한다
    • 파싱 도중에 중단하여 js파일을 실행하는 일이 없기 때문에 웹페이지 로딩이 빠르다
    • js파일을 여러개 선언하더라고 선언한 순서대로 실행되기 때문에 무작위 순서로 인해 발생하는 예상치 못한 오류를 줄일 수 있다

Use strict

  • Javascript는 초기에 빠르게 개발되었기 때문에 굉장히 유연하다
  • 유연함은 때때로 심각한 위험을 발생시킨다
  • ECMAScript 5에 추가
  • 바닐라 JS 사용 시 파일 최상단에 'use strict';를 선언
  • strict모드를 사용하면 위험도 줄일 수 있고 JS엔진의 효율적인 분석이 가능하여 성능 개선도 기대할 수 있다
